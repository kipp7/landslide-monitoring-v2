# 项目范围

这个仓库作为山体滑坡监测 V2 的公开源码树维护。当前 `main` 分支包含正在维护的桌面客户端、RK3568 边缘网关服务、RK2206 现场固件包、载板硬件交付资料、打包脚本和公开文档。

## 当前维护范围

- `apps/desktop-ui/` - React + Vite 监测操作界面。
- `apps/windows-shell/` - WPF + WebView2 Windows 宿主和安装器资源。
- `edge/rk3568-gateway/` - RK3568 边缘服务和部署模板。
- `firmware/rk2206-xl01/` - RK2206 XL01 现场节点固件包。
- `hardware/carrier-board/` - 载板设计和打板交付资料。
- `packages/` - 边缘服务共用的 TypeScript 包。
- `scripts/desktop/` - 桌面端开发、打包和验证脚本。
- `docs/` - 公开架构、发布、系统、维护和中英文文档。
- `.github/` - CI、Issue 模板、Pull Request 模板和依赖自动化配置。

## 公开边界

仓库刻意包含：

- 项目自有源码。
- 公开示例和部署模板。
- 可集成进兼容 vendor 构建树的固件包文件。
- 经筛选后适合公开复核的硬件交付文件。
- 构建、lint、打包和验证脚本。
- 公开项目治理文档。

仓库刻意排除：

- 生产后端服务和部署基础设施。
- 移动端应用和 Web Dashboard 应用代码。
- 完整供应商 SDK。
- 生成发布产物和本地构建输出。
- 真实凭据、私有端点、现场口令和本地环境文件。
- 客户特定点位配置、本地证据包和运行日志。
- 内部工作记录、本地笔记和本机运行状态目录。

## 维护规则

新增工作应让每个产品面保持明确。如果新增模块，应同时补充公开 README、职责边界、验证命令、安全复核，并在可行时纳入 CI。历史文件不应被视为仍在维护的 API、受支持模块或可发布资产，除非它们被重新放入明确的公开路径并补齐文档。
