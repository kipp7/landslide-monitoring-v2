# 通晓 SU03-T 静默固件决策

状态：已采用
日期：2026-07-20
适用范围：比赛与当前现场部署

## 决策

当前通晓告警终端不使用语音播报。现场告警仅使用蜂鸣器、振动马达、RGB 灯、板载独立报警灯和中文 LCD。

- RK2206 保持 `TONGXIAO_VOICE_ENABLED=0`，不初始化语音 UART，也不向 SU03-T 发送触发帧。
- 服务端保持 `TONGXIAO_VOICE_ENABLED=false`，不在 desired state 中启用语音输出。
- SU03-T 是独立上电的语音芯片，必须单独烧录静默固件；烧录 RK2206 的 `Firmware.img` 不能清除 SU03-T 自身播报。

## 静默版本

智能公元项目版本：`20260720001`

平台不允许零唤醒词、零命令词的工程通过配置检查，因此保留两个无输出占位项：

- 唤醒词：`静默维护终端`，低灵敏度，无唤醒回复。
- 命令行为：`SILENT_01`，命令词 `保持静默运行`，无回复、无触发项、无控制项。

其他配置：

- 开机播报：无。
- 退出回复与退出命令：无。
- 个性化音频：无。
- GPIO、UART、播放控制动作：无。

平台编辑页接受音量 `0`，但导出的版本 JSON 将该字段归一化为 `100`。静默保证依赖于工程中不存在任何播放内容、回复和控制动作，而不是依赖音量字段。烧录后仍必须执行冷启动实测。

```text

SHA256 F5B5862F9D19C28EA84EE37547FD0277B505F051316D57064924F3C45A2290FF
```

厂商串口升级工具使用：

```text
jx_su_03t_release_update.bin
SHA256 80FB50A37EA2F88B8E716F9CBD236D42D2B741D6ECE00C39B0434D59EB502AC6
```

固件包和导出的项目 JSON 不提交到公开仓库。项目 JSON 含平台凭证，只能保存在受控本地目录。

## 烧录与验收

1. 使用生产烧录工具时，点击 `BIN File` 并明确选择本版本的 `jx_su_03t_release.bin`。
2. 不得使用烧录工具目录自带的 `bin/uni_app_release.bin`；该文件是示例或工厂固件，可能包含开机播报，也不包含本项目配置。
3. 烧录完成后完全断电，再连续执行至少五次冷启动。
4. 每次启动均不得出现欢迎语、提示音或其它播报。
5. 无需向 RK2206 调试 Shell 发送 `AA 55 ...`；`OHOS #` 不解析 SU03-T 二进制帧，且当前 RK2206 语音功能已关闭。
6. 蜂鸣器、马达、RGB、独立报警灯、LCD、Wi-Fi 和 MQTT 告警功能必须保持正常。

如果静默固件烧录后仍有开机语音，优先核对烧录工具实际选中的 BIN 路径和 SHA256，不要通过修改 RK2206 固件规避问题。
