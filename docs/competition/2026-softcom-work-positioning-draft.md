# 2026 软通赛题作品定位草案

## 推荐作品名称

基于TX-SMART-R的山体滑坡智能预测预警与区域自适应多模态智防系统

## 备选名称

- 基于TX-SMART-R的山体滑坡数据驱动预测预警与区域自适应多模态智防系统
- 基于TX-SMART-R的山体滑坡数智融合预测预警与区域自适应多模态智防系统
- 基于TX-SMART-R的山体滑坡形变预测预警与视觉联动边云协同系统
- 基于TX-SMART-R的山体滑坡多源融合预测预警与数据链自愈监测系统

## 作品简介

本作品面向人工智能+地质灾害防治场景，针对山体滑坡监测预警中样本稀缺、数据分散、区域差异大导致模型与阈值难迁移，多源数据难统一判别，位移趋势与风险等级难提前识别，以及链路稳定性与数据可信度难保障等痛点，依托 TX-SMART-R 与 OpenHarmony/SwanLinkOS，构建覆盖边缘感知节点、边缘网关、云端分析平台和桌面/Web 可视化的山体滑坡智能防灾系统。作品以位移监测、位移预测和风险预警为核心，在既有地质形变监测、多传感采集、MQTT 上报、边云协同与可视化基础上，引入位移预测模型、预警判别模型、YOLO 视觉辅助识别模型和轻量化数据链健康模型，形成灾害体监测 + 数据链监测协同的多模型智能体系。其中，边缘协同层面向 RK3568 构建自托管边缘网关与智能体原生协同运行机制，具备多节点任务路由、状态记忆、健康摘要生成与本地快速响应能力，并部署轻量化边缘智检机制；同时融合不同地区公开数据、现场监测数据和系统运行数据，构建区域数据库 + 数据治理 + 区域模型库 + 模型匹配 + 持续训练闭环，实现按区域自适应训练、按场景智能匹配、按链路实时智检和按反馈持续优化。作品目标是推动滑坡监测系统从传统规则告警升级为面向人工智能+地质灾害防治、具备区域迁移、多模态融合、边缘智能和持续学习能力的新一代AI主动预测预警平台。

## 300字申报版

本作品面向人工智能+地质灾害防治场景，针对山体滑坡监测预警中事件样本稀缺、数据源分散且标签不足，不同区域诱因机制和演化特征差异大导致模型与阈值难迁移，多源监测数据难形成统一判据，以及位移趋势、风险等级难提前识别、现场链路稳定性与数据可信度难保障等问题，依托TX-SMART-R与OpenHarmony/SwanLinkOS，构建覆盖边缘感知、边缘网关、云端分析和桌面/Web的山体滑坡智能防灾系统。系统围绕位移监测、位移预测和风险预警，融合多传感采集、MQTT上报、边云协同、YOLO视觉辅助识别和轻量化数据链健康模型，形成“灾害体监测+数据链监测”双对象协同机制，可输出风险等级、预警建议与边缘侧快速响应结果；同时整合公开数据、现场监测数据和运行数据，构建区域数据库、模型库与持续训练闭环，实现按区域自适应训练、按场景智能匹配和按链路实时智检，推动滑坡监测由规则告警升级为AI主动预测预警平台。

## 历届相似作品竞赛态势判断

- 历届参赛数据库显示，报名类似作品共 16 个。
- 其中获得分赛区一等奖 2 个、二等奖 1 个、三等奖 4 个。
- 获得总决赛一等奖 1 个、二等奖 1 个、三等奖 0 个。

基于这组数据，可形成三个直接判断：

- 说明“山体滑坡/地质灾害监测预警”不是冷门题材，而是被历届赛事反复验证过、具有获奖可能性的有效方向。
- 也说明这一方向竞争并不稀缺，单纯做“传感采集 + 无线传输 + 平台展示 + 规则告警”的常规方案，很难再形成足够强的新意。
- 既然已有相似题材拿到总决赛一等奖，就不能再把“做出一个滑坡监测系统”当作核心亮点，必须把亮点前移到主动预测预警、区域自适应模型、多模态证据链、数据资产闭环和边缘智检能力上。

因此，今年的申报策略应明确为：

- 不把“题材本身”当创新，而把“技术代际升级”当创新。
- 不把“系统能跑”当亮点，而把“能提前判、能跨区域适配、能持续训练”当亮点。
- 不把“用了 AI”当卖点，而把“位移预测 + 风险判别 + YOLO 视觉辅助 + 数据链健康模型”的专用模型体系当卖点。

## 对比去年获奖作品的主要改进

| 对比维度 | 去年获奖作品主线 | 今年升级方向 |
| --- | --- | --- |
| 作品定位 | 以智能监测终端、微型气象站、边缘感知与展示预警为主 | 升级为“监测终端 + 区域级预测预警模型 + 多模态感知 + 边缘智检”一体化系统 |
| 核心能力 | 强项在多源采集、无线传输、平台展示、规则告警 | 强项前移到“形变位移预测 + 风险预警判别 + 模型匹配 + 智能联动” |
| 预警机制 | 以阈值触发和规则引擎为主，偏事后或准实时报警 | 升级为数据驱动的预测预警，强调提前量、动态阈值和低误报 |
| 数据来源 | 以单点现场监测数据为主 | 升级为“本地监测数据 + 不同地区公开数据 + 可扩展遥感/气象/地质数据”的多区域训练体系 |
| 模型体系 | 以轻量分析和规则判断为主，算法深度有限 | 自主训练位移预测模型和预警模型，形成区域模型库与模型匹配机制 |

| 系统可靠性 | 重点监测灾害体本身 | 新增“数据链健康监控”，同步监测传感器异常、通信异常、上传健康和链路退化 |
| 边缘智能 | 边缘侧以数据汇聚和初步判断为主 | 边缘侧部署轻量小模型，实现链路自检、本地告警、快速筛查与断网韧性 |
| 应用价值 | 偏单点示范、原型验证和功能展示 | 升级为可面向景区边坡、公路边坡、库区边坡、矿山边坡等场景复制推广的区域化方案 |
| AI 表达 | AI 主要体现在概念层和局部模块 | 今年可明确讲成“AI 元年下的主动预测、视觉认知、模型匹配、边缘智检”四位一体升级 |

## 建议用于答辩或申报的一句话定位

这不是一个只会采集数据的滑坡监测终端，而是一套能够按区域自适应训练、按场景智能匹配、按链路实时智检、按数据持续进化的山体滑坡智能预测预警系统。

## AI 落地表达

你们这套方案最适合的表达方式，不是泛泛地说“用了 AI”，而是直接讲清楚 AI 在系统里承担什么角色：

- 用位移预测模型，把实时监测数据转成未来趋势判断
- 用预警判别模型，把多源数据转成风险等级和预警建议
- 用 YOLO 视觉模型，为裂缝、坡表异常和落石风险提供辅助证据
- 用轻量化数据链健康模型，识别上传异常、链路退化和设备状态问题
- 在边缘协同层构建面向 RK3568 的自托管边缘网关与协同运行时，提升边缘侧任务编排、状态保持与快速响应能力
- 用区域数据库、数据治理、模型库和模型匹配机制，让系统具备持续训练和持续优化能力

## 可直接用于答辩的年度升级总结

如果说去年的作品解决的是“看得见、传得回、能报警”，那么今年的作品要重点回答“能不能提前判、能不能跨区域适配、能不能看懂现场、能不能监控自己的数据链是否可靠”。因此，今年的升级不只是多接一个摄像头或多做一个页面，而是把作品从“智能监测硬件原型”真正推向“具备算法壁垒和平台壁垒的智能预测预警系统”。这也是今年最容易打动评委的地方：你们不是简单延续去年，而是在原有终端、边缘网关和平台架构的基础上，补上了真正决定作品上限的核心技术层。

## 数据资产方向

今年建议把“数据资产”单独作为一条创新主线写出来。作品不只是训练一个模型，而是要建立面向山体滑坡场景的区域化数据库体系，包括区域滑坡基础库、多源监测时序库、遥感与视觉样本库、预警事件与人工标注库、设备与数据链健康库。通过统一编码、时间对齐、空间对齐、异常清洗、标签治理和数据版本化管理，逐步沉淀可清洗、可训练、可复用的滑坡行业数据底座。这样写的好处是，评委会看到你们不仅在做“一个功能”，而是在做“一个可以持续迭代的能力平台”。

## 创业化方向

这个项目完全可以往创业项目方向延展，商业逻辑也比较顺。硬件终端负责进入场景，边缘网关和平台负责承载业务，数据库和模型负责形成长期壁垒，后续再叠加运维服务、模型服务和区域复制服务。也就是说，作品的上限不只是比赛获奖，而是可以逐步发展成“感知终端 + 预警平台 + 数据中台 + 行业服务”的山体滑坡智能防灾产品。
