# RK2206 V1 载板原理图抄画稿

## 适用范围

这份文档只用于当前 `V1` 载板。

当前前提：
- 核心控制器仍然是小凌派 `RK2206` 开发板
- 外部 `DC` 电源模块已经自带保护和稳压
- 该电源模块可提供：
  - `12V`
  - `5V`
  - `3.3V`
- 当前实际要求：
  - `GPS` 供电使用 `5V`
  - `XL01 / SHT30 / MPU6050` 使用 `3.3V`

这版原理图的目标不是设计最小系统，而是：
- 去掉杜邦线
- 规范接口
- 保持当前已验证引脚真值

## 本版最重要的原则

1. 不把所有 `VCC` 混成一条网
2. 明确分成：
   - `5V_SYS`
   - `3V3_SYS`
   - `GND`
3. `GPS` 单独吃 `5V_SYS`
4. `XL01 / SHT30 / MPU6050` 吃 `3V3_SYS`
5. `RK2206` 开发板当前固定通过 `Type-C` 自身供电，载板不再反向给开发板送电
6. `I2C` 总线只工作在 `3.3V` 逻辑域
7. `UART` 引脚真值不改：
   - `PB2 / PB3` -> `XL01`
   - `PB6 / PB7` -> `GPS`
   - `PB4 / PB5` -> `I2C`

## 强提醒

当前仓内历史文档写过：
- `GPS VCC -> 3.3V`

但当前用户最新硬件真值明确改为：
- `GPS` 模块供电用 `5V`

因此本稿以当前用户最新口径为准。

但要注意：
- 当前仓内 `PINOUT.md` 与这份稿在 `GPS VCC` 上存在冲突
- 真正下板前，必须以你手头这只 GPS 模块的实际供电与串口规格为准

## 原理图总结构

```text
DC电源模块
├── 5V_SYS  ───────────────└── GPS模块VCC
│
├── 3V3_SYS ──────────────┬── XL01模块VCC
│                         ├── SHT30模块VCC
│                         ├── MPU6050模块VCC
│                         ├── I2C上拉电阻
│                         └── 去耦电容
│
└── GND ──────────────────┬── RK2206开发板GND
                          ├── GPS GND
                          ├── XL01 GND
                          ├── SHT30 GND
                          └── MPU6050 GND

RK2206信号
├── PB2 <──────────── XL01_TX
├── PB3 ─────────────> XL01_RX
├── PB4 ───────────── I2C_SDA ──┬── SHT30_SDA
│                               └── MPU6050_SDA
├── PB5 ───────────── I2C_SCL ──┬── SHT30_SCL
│                               └── MPU6050_SCL
├── PB6 <──────────── GPS_TX
└── PB7 ─────────────> GPS_RX
```

## 推荐元件清单

### 连接器

- `J_PWR`
  - 电源输入端子
  - 推荐 `1x3` 或 `1x4`
  - 至少引出：
    - `5V_SYS`
    - `3V3_SYS`
    - `GND`
- `J_XL01`
  - `1x4`
  - `3V3 / GND / RX / TX`
- `J_GPS`
  - `1x4`
  - `5V / GND / TX / RX`
- `J_SHT30`
  - `1x4`
  - `3V3 / GND / SDA / SCL`
- `J_MPU`
  - `1x4`
  - `3V3 / GND / SDA / SCL`
- `J_DBG`
  - 调试接口
  - 如果当前没有确认真实调试脚，就先只预留：
    - `3V3`
    - `GND`
    - `NC`
    - `NC`
  - 不要在本版强绑未确认调试脚
- `P_RK2206`
  - 这不是最终机械封装真值
  - 只是原理图中的“RK2206 开发板引出点”
  - 用来表达信号关系

### 电源和信号小器件

- `C1`
  - `100nF`
  - `0805`
  - `3V3_SYS` 主干去耦
- `C2`
  - `10uF`
  - `0805`
  - `3V3_SYS` 主干储能
- `C3`
  - `100nF`
  - `0805`
  - `5V_SYS` 去耦
- `C4`
  - `10uF`
  - `0805`
  - `5V_SYS` 去耦
- `R1`
  - `4.7k`
  - `I2C SDA` 上拉到 `3V3_SYS`
- `R2`
  - `4.7k`
  - `I2C SCL` 上拉到 `3V3_SYS`
- `C5`
  - `100nF`
  - `0805`
  - `XL01` 本地去耦
- `C6`
  - `10uF`
  - `0805`
  - `XL01` 本地储能
- `C7`
  - `100nF`
  - `0805`
  - `SHT30` 本地去耦
- `C8`
  - `100nF`
  - `0805`
  - `MPU6050` 本地去耦

### 机械

- `H1 ~ H4`
  - `M3` 安装孔

## 当前 GPS 串口口径

按你最新确认的硬件事实：

- 当前这只 `GPS` 模块自带串口电平保护
- 因此本版 `V1` 原理图按：
  - `PB6 <-> GPS_TX`
  - `PB7 <-> GPS_RX`
  直接连接处理

这意味着当前 `V1` 不再单独增加：

- `R3`
- `R4`
- `R5`

后续只有在更换成别的 GPS 模块时，才需要重新审一次串口电平兼容性。

## 原理图抄画版本

### 1. 电源入口

```text
J_PWR
--------------------------------
Pin1 -> 5V_SYS
Pin2 -> 3V3_SYS
Pin3 -> GND
--------------------------------
```

说明：
- 这里假设你的外部 DC 模块已经提供稳定 `5V` 和 `3.3V`
- 本板不再做二次降压

### 2. RK2206 开发板引出点

```text
P_RK2206
--------------------------------
Pin1  -> GND
Pin2  -> PB2
Pin3  -> PB3
Pin4  -> PB4
Pin5  -> PB5
Pin6  -> PB6
Pin7  -> PB7
--------------------------------
```

说明：
- 当前 `RK2206` 开发板自己通过 `Type-C` 供电
- 所以本版载板原理图不再给开发板送 `5V`
- 你在 EDA 里可以先把它当一个逻辑连接器来画
- 真正 PCB 时再按开发板排针/焊点/供电口实际结构落位
- `P_RK2206` 当前只表达：
  - `GND`
  - `PB2 / PB3 / PB4 / PB5 / PB6 / PB7`

### 3. XL01 接口

```text
J_XL01
--------------------------------
Pin1 -> 3V3_SYS
Pin2 -> GND
Pin3 -> XL01_TX_TO_MCU
Pin4 -> XL01_RX_FROM_MCU
--------------------------------
```

连接关系：

```text
J_XL01.Pin3 (XL01_TX) -> P_RK2206.Pin2 (PB2 / UART2_RX_M1)
P_RK2206.Pin3 (PB3 / UART2_TX_M1) -> J_XL01.Pin4 (XL01_RX)
J_XL01.Pin1         -> 3V3_SYS
J_XL01.Pin2         -> GND
```

### 4. GPS 接口

```text
J_GPS
--------------------------------
Pin1 -> 5V_SYS
Pin2 -> GND
Pin3 -> GPS_TX
Pin4 -> GPS_RX
--------------------------------
```

连接关系：

```text
J_GPS.Pin1 -> 5V_SYS
J_GPS.Pin2 -> GND
J_GPS.Pin3 (GPS_TX) -> P_RK2206.Pin6 (PB6)
P_RK2206.Pin7 (PB7) -> J_GPS.Pin4 (GPS_RX)
```

### 5. SHT30 接口

```text
J_SHT30
--------------------------------
Pin1 -> 3V3_SYS
Pin2 -> GND
Pin3 -> I2C_SDA
Pin4 -> I2C_SCL
--------------------------------
```

### 6. MPU6050 接口

```text
J_MPU
--------------------------------
Pin1 -> 3V3_SYS
Pin2 -> GND
Pin3 -> I2C_SDA
Pin4 -> I2C_SCL
--------------------------------
```

### 7. I2C 总线公共连接

```text
P_RK2206.Pin4 (PB4) -> I2C_SDA
P_RK2206.Pin5 (PB5) -> I2C_SCL

I2C_SDA -> J_SHT30.Pin3
I2C_SDA -> J_MPU.Pin3
I2C_SDA -> R1 -> 3V3_SYS

I2C_SCL -> J_SHT30.Pin4
I2C_SCL -> J_MPU.Pin4
I2C_SCL -> R2 -> 3V3_SYS
```

### 8. 3.3V 去耦

```text
3V3_SYS -> C1 -> GND   (100nF)
3V3_SYS -> C2 -> GND   (10uF)
```

建议位置：
- 靠近 `J_SHT30 / J_MPU / I2C` 区域

### 8.1 XL01 本地去耦

```text
J_XL01.Pin1 (3V3_SYS) -> C5 -> GND   (100nF)
J_XL01.Pin1 (3V3_SYS) -> C6 -> GND   (10uF)
```

建议位置：
- 紧贴 `J_XL01` 电源脚附近
- 优先靠近 `XL01` 接口一侧

### 8.2 SHT30 本地去耦

```text
J_SHT30.Pin1 (3V3_SYS) -> C7 -> GND   (100nF)
```

建议位置：
- 紧贴 `J_SHT30` 电源脚附近

### 8.3 MPU6050 本地去耦

```text
J_MPU.Pin1 (3V3_SYS) -> C8 -> GND   (100nF)
```

建议位置：
- 紧贴 `J_MPU` 电源脚附近

### 9. 5V 去耦

```text
5V_SYS -> C3 -> GND    (100nF)
5V_SYS -> C4 -> GND    (10uF)
```

建议位置：
- 靠近 `J_GPS`

### 10. 开发板与载板供电关系

```text
RK2206开发板:
- 自己通过 Type-C 供电

载板:
- 自己从外部 DC 模块拿 5V_SYS / 3V3_SYS

两边必须:
- 共地
```

说明：
- 本版不再从载板给 `RK2206` 开发板送电
- 但 `P_RK2206.Pin1 (GND)` 必须和载板 `GND` 相连
- 否则 UART / I2C 参考地不一致，通信会不稳定或直接失效

## 你在 EDA 里要画的网络名

- `5V_SYS`
- `3V3_SYS`
- `GND`
- `XL01_TX_TO_MCU`
- `XL01_RX_FROM_MCU`
- `I2C_SDA`
- `I2C_SCL`

## 当前不建议你在本版加入的内容

- 不要加 `12V` 到板上其它地方
- 不要加 `RS485`
- 不要加额外传感器接口
- 不要强绑未确认的调试 TX/RX
- 不要把 `5V_SYS` 和 `3V3_SYS` 混成同一网

## 抄画顺序

1. 先画 `J_PWR`
2. 再画 `P_RK2206`
3. 再画 `J_XL01 / J_GPS / J_SHT30 / J_MPU`
4. 再画 `R1 / R2`
5. 再画 `C1 / C2 / C3 / C4`
6. 最后统一检查网名是否连对

## 最后检查清单

- `GPS` 供电是不是接在 `5V_SYS`
- `XL01 / SHT30 / MPU6050` 是不是接在 `3V3_SYS`
- `PB2/PB3/PB4/PB5/PB6/PB7` 是否完全没改
- `I2C` 两个上拉是否都拉到了 `3V3_SYS`
- `5V_SYS` 和 `3V3_SYS` 是否没有误短接
- `GPS_TX / GPS_RX` 是否按当前模块直连到 `PB6 / PB7`
- `RK2206` 开发板是否确认继续独立 `Type-C` 供电
- `RK2206` 与载板是否已经可靠共地
- `C5 / C6` 是否紧贴 `J_XL01`
- `C7` 是否紧贴 `J_SHT30`
- `C8` 是否紧贴 `J_MPU`
