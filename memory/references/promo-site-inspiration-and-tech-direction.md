# Reference: promo-site-inspiration-and-tech-direction

## Purpose

Store the current inspiration set, trend references, and technical direction for the project's future promotional or technology website so later sessions do not need to reconstruct the same design research from chat logs.

## 2026 Trend Alignment

- The current direction is aligned with 2026 high-visibility web design trends:
  - `3D / immersive elements`
  - `experimental navigation`
  - `vibrant palettes`
  - `maximalist layouts`
- The current development constraint is:
  - keep the site visually ambitious
  - but avoid turning it into a heavy, JS-only experience with poor server-first performance

Primary references:

- Figma 2026 Web Design Trends
  - https://www.figma.com/resource-library/web-design-trends/
- Figma 2026 Web Development Trends
  - https://www.figma.com/resource-library/web-development-trends/

## Website Direction

Current recommended positioning:

- not a standard enterprise website
- not a pure docs portal
- but a hybrid of:
  - brand site
  - technical credibility site
  - immersive scenario-driven digital theater

Working one-line concept:

- turn the landslide monitoring system into a future-facing risk command theater that visitors can enter, watch, and explore

## Current Reference Split

Current hybrid baseline is now clearer:

- primary structure reference:
  - `Hut 8`
- secondary scene / interaction reference:
  - `Lusion`

What this means:

- use `Hut 8` for:
  - homepage chapter order
  - repeated pinned-stage grammar
  - infrastructure / platform storytelling rhythm
- use `Lusion` for:
  - fixed canvas + DOM overlay stagecraft
  - giant editorial type moments
  - one concentrated immersive chapter
- do not use `Lusion` for:
  - whole-homepage information architecture
  - agency portfolio wall behavior
  - full-site custom scroll as the default

## Visual Direction

Preferred visual language:

- dark
- mineral / geological
- industrial safety
- command-center futurism

Avoid:

- generic purple AI glassmorphism as the main identity

Recommended core visual object:

- `digital mountain body + risk pulse network`

Alternative primary motifs:

- digital mountain twin core
- risk pulse sphere
- geological cross-section volume
- monitoring network neural core

## Structure Direction

Recommended homepage chapter flow:

1. Hero
2. Problem framing
3. Sensing network
4. Data and platform
5. Command and alerting
6. Technical credibility layer
7. Final CTA

## Tool Positioning: `three-vue-tres`

Source:

- repository:
  - https://github.com/hawk86104/three-vue-tres/tree/master
- package:
  - https://github.com/hawk86104/three-vue-tres/blob/master/package.json

Current interpretation:

- strong candidate as:
  - scene lab
  - immersive demo page engine
  - concept proof environment
- not recommended as:
  - the full outer shell of the promotional site

Reason:

- it is better understood as a 3D application framework
- the outer site still needs:
  - brand storytelling
  - information rhythm
  - SEO / content delivery
  - fine-grained performance control

Current recommended architecture split:

- outer site:
  - `Next.js 16 + React 19 + Tailwind CSS v4 + Motion`
- heavy scene layer:
  - `Three.js / WebGPU-first`
  - or `three-vue-tres` as an isolated immersive module
