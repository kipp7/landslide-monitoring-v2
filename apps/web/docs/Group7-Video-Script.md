# Group 7 演示视频脚本（按“现状→问题→方案→架构→模块”结构）

建议1080p，30fps；两套时长：120s 与 150s。结构统一：山体滑坡现状→传统系统问题→我们设计的系统→整体架构→逐模块介绍→收尾。

## 120s（标准版）
- 0:00–0:08 山体滑坡现状
  - 旁白：山体滑坡一直以来都是山地与雨季高发的重大自然灾害，预警与响应每一秒都至关重要。
- 0:08–0:18 传统系统问题
  - 旁白：过去的系统往往界面分散、图表割裂、编码不统一，多源数据格式不一、刷新不同步，导致识读困难、协同低效。
- 0:18–0:26 我们的系统与目标
  - 旁白：为了解决这些问题，我们设计了一套“山稳数境（Mountain Insight）”系统，用统一的视觉与交互整合数据，突出重点，优化从发现到处置的全链路。
- 0:26–0:36 整体架构（总览）
  - 旁白：系统采用前后端一体化的分层架构：前端 Next.js App Router 与 API Routes，后端业务服务，数据层 PostgreSQL/Realtime，IoT 接入入库与推送。
- 0:36–0:50 模块一：设备管理
  - 旁白：本段重点介绍“监测站管理”和“基准点管理”。设备状态监控仅用于快速筛选与状态浏览；在监测站管理（MonitoringStationManagement）中，我们按区域与站点查看清单与在线率，结合近况与故障概况进行台账维护，支持关键字搜索与分页浏览，并可新增/编辑站点信息（名称、类型、位置与备注等）；从站点项可一键跳转到对应设备详情，形成从站点到设备的闭环排查；在基准点管理（BaselineManagementV2）中，我们一键建立/重建基准点，查看拟合度、残差分布与可信区间，并结合质量评估结果；支持按设备与时间窗切换、参数微调与结果回看，并与地质形变监测互相跳转以核验曲线，最终将通过评估的结果保存为站点长期对照基线。
- 0:50–1:06 模块二：GPS监测
  - 旁白：在地质形变监测中，我们先用设备与时间范围选择器定位目标数据，必要时开启自动刷新；顶部风险卡片基于国标四级预警显示当前等级，数据质量面板提供置信度与质量评分。实时分栏包含三类核心视图：位移趋势（总位移/水平/垂直，单位毫米，支持智能刻度与高亮）、形变速度（mm/h，支持平滑与范围智能扩展）、环境因素关联（温度/湿度双轴对照，提示潜在气象驱动）；并配合“数据点数”与“设置”实现演示/分析两种节奏的切换。分解分栏提供 CEEMD 能力：展示 IMF 分量时域图（多曲线对比）、主频谱柱状图（IMF1~IMF n 主频特征）、长期趋势（残差分量）与能量分布（各分量能量占比），页面同时给出分量数、信号长度与分解质量等指标，便于识别噪声、周期与长期漂移。预测分栏使用增强预测组件：短期（未来24小时）与长期（7天）曲线并列，历史窗口支持自适应/固定/智能，平滑过渡可开关；标题区展示算法名与置信度，右侧提供“预测误差评估”图（1h/6h/12h/24h/3d/7d），当后端返回模型性能（MAE/R²/置信度）时以真实误差与时间衰减计算误差条；“风险预警分析”面板联动显示当前预警等级与关键因子（最大位移、模式相似度、必要时包含实时基线位移），并保留原始因子 JSON 以备核验。数据详情分栏包含统计概览（数据量、最大/平均位移、时间跨度）、位移分布直方图（动态区间）与可分页/排序的明细表；工具区支持导出“分析结果报告”“综合监测报告”，并提供“强制刷新”以清缓存重算。额外说明：图例筛选与悬浮提示支持查看单点时间/数值/单位；可从页面跳转至“基准点管理”进行参数校准后回看预测效果；“设置”中的数据点数限制附带推荐范围（100–500条）以平衡性能与信息量。
- 1:06–1:20 模块三：基线管理
  - 旁白：基线管理支持一键建立与质量评估，输出拟合度与可信区间等关键指标；页面与地质形变监测和设备管理互相跳转，便于回溯参数并持续优化监测策略。
- 1:20–1:34 模块四：分析与系统监控
  - 旁白：在分析页，我们提供“异常类型分布”和国标四级预警统计，基于过去24小时窗口定时刷新，并在提示中给出总计、占比与最近发生时间；同时，同屏展示实时异常表与监测地图，便于我们按时间顺序查看告警并快速定位到对应设备与区域。在系统监控中，我们提供缓存命中率趋势和实时消息流量图，显示连接状态与客户端统计，并支持一键清空/预热缓存与刷新聚合数据，且可切换自动/手动刷新与刷新间隔。
- 1:34–1:46 设计亮点
  - 旁白：整个系统在信息层级、色彩编码与对比关系上保持一致，关键任务路径三步可达，降低识读与操作负担。
- 1:46–2:00 收尾
  - 旁白：总结而言，山稳数境用“可视化+交互”打通选择→感知→分解→预测→核验→导出的全链路，显著缩短从发现到处置的时间。系统已适配 PC 与大屏，支持演示/实战两种节奏；后续将联动告警处置闭环与移动端值守，使研判更快、联动更顺。以设计守护坡体安全，让重要信息更快抵达决策。谢谢观看。

## 150s（加长版）
- 0:00–0:08 山体滑坡现状
  - 旁白：山体滑坡一直以来都是高影响、强不确定性的自然灾害，特别在复杂地形与雨季，早期识别对降低风险至关重要。
- 0:08–0:20 传统系统问题
  - 旁白：过去的系统层面上，多平台与多模块割裂明显；呈现层面上，图表堆叠、编码不统一；数据层面上，多源数据分散、刷新频率不一。这些问题综合导致识读慢、判断难与复盘弱。
- 0:20–0:30 我们的系统与目标
  - 旁白：为了解决这些问题，我们设计了“山稳数境（Mountain Insight）”。它以统一视觉语言与交互流程为核心，使关键信息一眼可见，并显著缩短从监测到研判的链路。
- 0:30–0:44 整体架构（总览）
  - 旁白：系统采用前后端一体化的分层架构：前端 Next.js App Router 与 API Routes，后端业务服务，数据层 PostgreSQL/Realtime，IoT 接入入库与推送。
- 0:44–1:02 模块一：设备管理
  - 旁白：本模块以“监测站管理”和“基准点管理”为核心，设备状态监控保持精简用于快速筛选与状态浏览；在监测站管理（MonitoringStationManagement）中，我们按区域/站点维护清单，查看站点在线率、近况与故障汇总，支持关键字搜索、分页与新增/编辑站点属性字段，并可从站点一键跳转至对应设备/详情以闭环定位；在基准点管理（BaselineManagementV2）中，我们一键建立或重建基准点，查看拟合度、残差分布、可信区间与质量评分，支持按设备与时间窗切换、参数微调、结果回看；页面与地质形变监测/设备管理双向导航，支撑参数回溯、曲线核验与策略迭代优化。
- 1:02–1:22 模块二：GPS监测
  - 旁白：该模块完整走通“选择→感知→分解→预测→核验→导出”的闭环。我们通过设备与时间范围选择器进入目标数据并可开启自动刷新；实时分栏集中呈现位移/速度/环境三视角，配合国标预警卡片与数据质量面板形成“当前状态+变化态势”的组合；CEEMD 分栏给出 IMF 时域、IMF 主频谱、长期趋势与能量占比，并对外显示分量数、信号长度与分解质量，帮助识别周期、噪声与缓慢漂移；预测分栏以增强组件展示短/长期预测，并允许配置历史窗口与曲线平滑，旁侧以1h→7d的误差评估条回溯模型可信度，同时在“风险预警分析”中解释风险等级与关键因子；数据详情分栏包含统计、直方图与明细表，并配套“导出分析/综合报告”“强制刷新”。此外，“设置”提供数据点数限制调节，保障在演示与深度分析间取得性能与信息量的平衡。补充：支持图例筛选和悬浮提示（时间/单位/数值）；可从该页跳转到“基准点管理”进行参数校准与曲线核验；数据点数限制提供推荐区间（100–500条）。
- 1:22–1:38 模块三：基线管理
  - 旁白：基线管理提供一键建立与质量评估，产出拟合度、残差分布与可信区间等指标；支持与地质形变监测/设备管理之间的往返导航，方便回溯参数、比对效果并迭代优化基线设置。
- 1:38–1:54 模块四：分析与系统监控
  - 旁白：分析页将异常类型分布与四级预警统计联动呈现，按照24小时窗口定时更新，并提供总计、占比与最近发生时间等要素；实时异常表与地图同屏显示，帮助我们更直观地跟踪告警与定位设备。在系统监控中，我们查看缓存命中率趋势、实时消息流量与连接状态，结合客户端统计判断系统运行状况；同时，页面提供清空/预热缓存与刷新聚合数据的快捷操作，并支持自动刷新开关与刷新间隔切换，便于运维与演示。
- 1:54–2:10 设计亮点
  - 旁白：我们统一信息层级、色彩编码与对比关系，确保跨页面的一致识读体验，同时兼顾性能与稳定性，保障长时运行下的流畅呈现。
- 2:10–2:30 收尾
  - 旁白：总结提升：山稳数境以统一的视觉语言、清晰的任务链路和数据产品化能力，显著提升监测识读效率与跨角色协同深度；可落地于易发区的日常值守、会商研判与应急演练，支持 PC/大屏扩展与 IoT 接入演化。下一步我们将强化联动处置闭环、移动端巡检与模型在线评估，以设计驱动预警能力持续进化，守护坡体与社区安全。谢谢观看。
