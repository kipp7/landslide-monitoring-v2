## Why

当前 `apps/desk-win` 只是“能跑起来”的 WebView2 壳，距离可交付的 Win11 原生软件还有差距：缺少系统托盘、通知、开机自启、深链接、原生文件对话框等“桌面软件必备体验”。

本变更的目标是把桌面端从“网页套壳”提升为“可落地的 Windows 桌面软件”，并让这些原生能力通过稳定的 Host Bridge 暴露给 `apps/desk` 使用。

## What Changes

- 增加系统托盘（最小化到托盘、右键菜单、快速操作）
- 增加 Windows 原生通知（Toast）能力，并提供开关与权限提示
- 增加可选的开机自启（可在设置中开关）
- 增加 `landslide://` 协议深链接（从外部唤起并跳转到指定路由）
- 增加原生文件对话框集成（导入/导出、保存截图/日志等）
- 增加“诊断与日志”入口（便于交付现场排障）

## Non-Goals

- 不做后端 API 修复与联调（本阶段继续 Mock-first）
- 不引入复杂的原生 UI 框架替换（仍保持 WPF + WebView2）

## Impact

- 需要引入少量桌面端依赖（托盘/通知/安装注册表项等）
- 增加 Host Bridge 事件与动作类型，需要前端配合调用
- 交付体验提升：用户可像普通桌面软件一样使用、最小化、后台驻留与接受通知
