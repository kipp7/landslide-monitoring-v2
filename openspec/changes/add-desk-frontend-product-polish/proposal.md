## Why

目前 `apps/desk` 已具备主要页面与 Mock 数据能力，但要达到“可以交付演示/内测”的产品级体验，还需要补齐 UI 细节、可读性、交互一致性与可用性边界（空状态、加载状态、错误状态、键盘快捷键等）。

本变更聚焦于 **不改变业务范围** 的前提下，把 UI 做到更精致、更稳定、更像真实产品。

## What Changes

- 统一产品级文案与品牌表达（标题、副标题、提示语更像可交付产品）
- 修复暗色主题对比度问题（避免黑字叠深色背景）
- 补齐加载/空/错误状态（含 Skeleton/占位）
- 增加键盘可用性（Esc 关闭弹窗、Enter 提交、常用快捷键）
- 增加一致的页面标题区、面包屑/返回路径与操作区布局
- 增加“桌面端模式”识别（在桌面壳里显示更多原生入口，如“退出软件”）

## Non-Goals

- 不对齐所有后端接口，不做真实数据联调
- 不引入重型 UI 框架替换
